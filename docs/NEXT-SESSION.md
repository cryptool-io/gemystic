## 1. Non-negotiable working rules (learned this session)

1. **The em dash character is banned everywhere.** Every user-facing string, doc,
   commit message, generated copy, dataset. Purged once already; do not reintroduce.
2. **Never run `next build` while `next dev` is running.** They share `.next` and
   corrupt each other. Stop the preview server, `rm -rf .next`, build, restart.
   This bit us twice.
3. **All colours and fonts go through the semantic tokens** (the `@theme` block in
   `app/globals.css`, `--font-plex` / `--font-display`). No hex literals, no inline
   styles. Brand green = positive, accent amber = caution; never flatten the two.
   ESLint warns on violations (no-restricted-syntax, same rule as Trust-Agent).
4. **Verify in the browser, not just typecheck.** The owner checks screenshots.
   Test at 375px and 1440px minimum. The in-app browser pane has
   `prefers-reduced-motion: reduce` active, so the marquee test shows via
   `getComputedStyle(track).animationName`, not visually.
5. **Money maths**: USD is the only stored currency; convert at display via
   `lib/currency.ts` only. Discounts round per item, then sum. Never floats
   in new persistence, `NUMERIC(12,2)`.
10. **Stack parity with Trust-Agent is a standing owner requirement.** Same
   architecture, stack and conventions as C:\Users\ronza\Claude\Trust-Agent:
   Next 16 + React 19.2, root-level app/ components/ lib/ (no src/), Tailwind 4
   CSS-first, Prisma 6 with the lib/prisma.ts singleton, ESLint 9 flat config,
   node --test via tsx, native Postgres (no Docker). Do not diverge.
6. **The owner expects the request tracker maintained.** Append a turn section to
   docs/REQUEST-TRACKER.md every working session, honestly graded.
7. **Heredoc gotcha**: bash heredocs containing JS template literals break; write
   python patch files to the scratchpad and run them instead.
8. **Etsy fetches 403 from scripts** (TLS fingerprinting). Use the in-app browser
   to collect listing ids (sessionStorage accumulator pattern in the transcript)
   or the `--from-files` mode of scripts/etsy-sync.mjs.
9. Git identity/user data: `var/` is gitignored runtime state (accounts, sessions,
   reviews, campaigns, sold, interest, settings, outbox, backups). Never commit it.
   `SECURITY-gemysticgems.com.md` stays local-only (documents a live compromise).

## 2. Current architecture in one paragraph

Next.js 16 App Router (Trust-Agent stack parity: React 19.2, Tailwind 4 CSS-first
via the `@theme` block in app/globals.css, ESLint 9 flat config, root-level
app/ components/ lib/ layout), static-first (231 pages), catalogue generated by
`npm run normalize` from `data/etsy-raw.txt` + `data/species.json` +
`data/taxonomy.json` into `data/catalog.json`. Runtime mutable state lives in
JSON stores under `var/` behind driver-style modules (auth, reviews, campaigns,
sold, interest, settings), each shaped 1:1 with a Postgres table in
`prisma/schema.prisma`, live on native PostgreSQL 17 (database `gemystic`, all
tables migrated; stores not yet swapped onto Prisma). Infra adapters:
mailer (file|smtp|ses), storage (local|s3), backups (local|s3), AI
(anthropic|bedrock|openai-compatible chain in `lib/ai.ts`). Auth is bcrypt +
opaque hashed session cookies, RBAC customer<staff<admin<owner, first registered
account becomes owner. Admin under `/admin`, internal tooling under `/studio`.

## 4. Milestones (work in this order)

### M1: Database live + auth/store migration
Prereq: none (B1 completed: schema is migrated and live on native Postgres 17).
1. DONE: database `gemystic` exists with all tables; verify anytime with
   `npm run db:studio`.
2. DONE: auth store runs on Prisma when DATABASE_URL is set (PrismaUserStore in
   lib/auth/store.ts; JSON driver remains the keyless fallback). Existing
   accounts imported via `npm run db:import-stores` (idempotent; JSON files kept
   in place as rollback, no longer read).
3. DONE for reviews + campaigns (Prisma drivers behind the same functions; new
   campaigns table + reviews.product_slug in migration
   20260718213829_m1_reset_tokens_campaigns_review_slug; seed reviews auto-seed
   an empty DB on first read). STILL JSON: sold + settings, their reads are
   synchronous inside the catalogue query path; swapping them means an async
   refactor through lib/catalog.ts, do it deliberately, not as a drive-by.
4. DONE: password reset live end-to-end: password_reset_tokens table (hashed,
   single-use, 60 min TTL, re-request invalidates), /forgot + /reset/[token]
   pages, throttled /api/auth/forgot + /api/auth/reset, mail via mailer(),
   all sessions revoked on success. Verified: request → outbox email → reset →
   old password rejected → token reuse rejected.
5. Acceptance (verified 18 July 2026 against local Postgres): register/login/
   roles work off the DB, test user landed in users table with role customer,
   JSON file untouched; reset email lands in var/outbox in dev.
